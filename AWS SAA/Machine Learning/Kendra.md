Amazon Kendra is a **fully managed, ML-powered enterprise search service** that lets users **search across documents and get direct answers** instead of just a list of links.

---

## What it does

- Searches across large collections of documents
- Uses **machine learning + natural language understanding**
- Returns **direct answers from content**, not just results

Think of it as:

> “Google, but for your company’s internal documents”

---

## Supported Data Sources

Amazon Kendra can index documents from:

- PDF files
- Word documents
- PowerPoint files
- HTML pages
- FAQs
- Text files
- Enterprise data sources (various AWS and external systems)

---

## How it works

### 1. Data ingestion

- Documents are collected from multiple sources

### 2. Indexing

- Kendra builds a **search index**
- Extracts meaning using ML (not just keyword matching)

### 3. Querying

- Users ask questions in **natural language**
- Example:
    - “Where is the IT support desk?”

### 4. Answer retrieval

- Returns **direct answer**
    - Example: “1st floor”

---

## Key Features

### 1. Natural Language Search

- Understands user intent (like Google search)
- Not limited to keyword matching

---

### 2. Answer Extraction

- Extracts exact answers from documents
- Instead of just showing document links

---

### 3. Relevance Learning (Feedback-based improvement)

- Learns from user behavior
- Improves ranking over time

This is called:

- **Incremental learning**

---

### 4. Search Tuning

You can improve results using:

- Data freshness
- Content importance
- Custom ranking rules
