Amazon Polly is a **text-to-speech (TTS)** service that uses **deep learning** to convert text into lifelike speech.

- Opposite of **Amazon Transcribe** (speech → text)
- Converts **text → audio**
- Enables applications to “talk”

---

## Key Features

### 1. Text-to-Speech

- Input: Text
- Output: Natural-sounding speech (audio)

### 2. Neural Voices

- Uses **neural networks** for more realistic, human-like speech

---

## 2. Customization Features

### 2.1 Pronunciation Lexicons

- Used to **control how specific words are pronounced**
- Helpful for:
    - Stylized words (e.g., “St3ph4ne” → “Stephane”)
    - Acronyms (e.g., “AWS” → “Amazon Web Services”)

### Use Case:

- Upload lexicon file → apply during speech synthesis

---

### 2.2 SSML (Speech Synthesis Markup Language)

- Allows **fine control over speech output**

### You can:

- Add pauses (breaks)
- Emphasize words
- Control tone and pitch
- Use phonetic pronunciation
- Add whispering or breathing effects
- Use speaking styles (e.g., newscaster)

### Example Capabilities:

- Pause between sentences
- Emphasize keywords
- Customize delivery style

---

## 3. Use Cases

- Voice assistants
- Accessibility (reading text aloud)
- Automated announcements
- E-learning narration
- Customer service bots

---

## Key Points

- Polly = **Text → Speech**
- Uses **deep learning** (neural voices)
- Two key customization tools:
    - **Lexicons** → control pronunciation
    - **SSML** → control how speech sounds
- Often paired with:
    - Transcribe (Speech → Text)
